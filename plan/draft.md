링크: https://b.kakao.com/views/PlayMCP/AGENTIC_PlAYER_10



"나챗방" 이라는 MCP에 감명을 받아서 진행하려고 하고 있어.



내 컨셉은 다음과 같아.



"기가채드"와 관련된 국내 밈 중에서 "내면의 기가채드" 라는 밈이 있어.

내가 주눅들고 힘이 들 때에도 내면의 기가채드는 나를 응원하며 북돋아준다는 밈이야.

링크: https://namu.wiki/w/%EA%B8%B0%EA%B0%80%EC%B0%A8%EB%93%9C#s-2.1



이것처럼 "나와 대화하기" 기능에서 활용할 수 있는 MCP를 만들고 싶어.



보통 "나와 대화하기"기능은 단순 메모용 (유용한 정보가 있는 링크, 관심 있는 상품 목록 및 링크, API KEY 등)으로 자주 사용한단 말이지?



이를 활용하고 싶은거야.



내가 구현하고자 하는 주요 기능은 다음과 같아.

1. 사용자의 메모를 "용도"별로 구분하여 저장

- "용도"는 큰 카테고리로 지정되며, 채팅 형태로 사용자만의 카테고리를 추가할 수 있음

- 해당 카테고리에 있는 정보는 context로 활용될 수 있으며, 사용자의 질문에 응답할 때 능동적으로 사용 할 수 있음. 예를 들면, 관심있는 상품 목록들 danawa 링크들을 모았을 때, 해당 상품들을 상품군별로 구분하고 사용자가 "나 모니터 뭘 살지 모르겠어" 라고 질문했을 때, 해당 상품들을 context로 활용하여 응답에 도움을 줄 수 있음.

- 기본적인 "용도" 카테고리는 다음과 같음: 일정, 연락처, 파일, 장소(네이버 지도 링크 등), URL

2. 사용자는 해당 context 기반으로 질의응답을 할 수 있음. 예시는 다음과 같음.

- "오늘 일정 정리해줘" 라는 사용자 질의가 들어오면, 에이전트는 "채팅 내역"에서 정리한 데이터를 토대로 현재 시각(UTC +9) 기반으로 현재 진행중인 일정 + 앞으로의 일정을 제출하고, 내일 일정, 1주일 일정도 정리해서 보여줌

- "URL" 데이터의 경우, 에이전트에 웹 mcp 혹은 tool을 사용하여 해당 URL에 어떤 정보가 있는지 파악하고 요약-정리 할 수 있도록 함. 그 이후, URL에 대한 세부 분류를 에이전트 판단 하에 정리함. 디폴트 URL 분류값은 다음과 같음: 장소(지도), 정보, 상품

3. 사용자와 친밀한 대화도 진행 가능함

- 말투의 경우는 "내면의 기가채드" 스타일을 반영함. 해당 스타일에 대해 분석하고 프롬프트를 추가적으로 작성할 필요가 있음.

- 사용자를 지칭하는 명칭은 "삣삐". 예를 들어, Q. "기가채드 거기 있어?" -> A. "오브콜스, 삣삐."
